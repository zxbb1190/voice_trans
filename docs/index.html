为游戏语音场景做的实时字幕工具

它不是通用会议软件，而是围绕 PC 游戏、Discord 队友沟通、海外服开黑和直播字幕辅助设计的轻量桌面工具。

🔊

系统声音捕获

通过 WASAPI Loopback 捕获电脑正在播放的声音。首次向导和设置窗口都能显示音量条，先确认真的听到了游戏或 Discord。

🧠

本地语音识别

基于 faster-whisper 离线识别语音，当前可固定英语或中文识别，Lite 版首次运行下载模型。

🌐

中英双向翻译

当前正式支持英语 → 中文、中文 → 英语。其它语言暂未作为可选语言开放。

🪟

游戏浮窗

透明置顶字幕窗口，支持拖拽、调整透明度、显示原文和译文，不需要切出游戏查看翻译。

📱

手机同步

同一局域网下用手机浏览器打开页面，即可实时接收字幕，适合把手机当副屏使用。

🛠️

反馈诊断模板

调试模式会记录最近一次识别、翻译和浮窗更新延迟。提交反馈时会自动生成带版本、设备、服务商和日志路径的模板。

选择适合你的下载版本

第一次体验建议下载 Lite 版。解压运行后先走首次启动向导，确认 API Key 和音频设备都可用；网络环境较差时可以选择 Full 版。

五步完成玩家首次闭环

先确认接口和音频能跑，再进入游戏。遇到问题时，用内置反馈模板把诊断信息一次带全。

下载并解压

下载 Lite 或 Full 包，解压后运行 VoxGo.exe。

测试 API Key

首次向导里选择翻译服务，填写 API Key、模型名和地址，然后点击测试，确认接口真实可用。

测试系统声音

选择 [系统声音] 或 Loopback，播放游戏、Discord 或视频声音，确认音量条会跳动。

进入游戏测试

播放游戏、Discord 或视频语音，浮窗会显示原文和翻译结果。

反馈诊断信息

如果卡住，开启调试模式并点击提交反馈，复制包含设备、服务商、延迟和日志路径的模板。

常见问题

把用户最担心的问题先说清楚，可以减少误解，也方便你收集更高质量的测试反馈。

它会录我的麦克风吗？

默认目标是捕获 Windows 系统声音输出，不是麦克风。设置里应优先选择 [系统声音] / Loopback 设备；普通麦克风通常录不到游戏声音。

会不会注入游戏进程？

不会。它是独立桌面程序，通过系统声音设备采集音频，并用透明置顶窗口显示字幕，不修改游戏客户端。

为什么没有字幕？

通常是音频设备没选对，或者游戏 / 微信 / Discord 声音走了另一个输出设备。先在向导或设置里测试音频，看音量条是否跳动，再确认 Windows 音量混合器里的输出设备和工具里选择的 Loopback 设备一致。

为什么识别不准？

真人语音比视频更难识别，尤其是短句、口音、噪声、背景音乐和压缩音频。可以尝试提高模型大小、降低游戏背景音乐、固定识别语言，并反馈样例。

支持哪些语言？

当前正式支持英语和中文双向：英语 → 中文、中文 → 英语。其它语言暂未在界面和识别/翻译流程中开放为可选项。

Lite 和 Full 有什么区别？

Lite 包更小，首次运行会下载 Whisper 模型；Full 包更大，内置 Whisper small 模型，适合网络不稳定的用户。

支持哪些游戏？

只要游戏语音能从当前扬声器、耳机、HDMI、USB 声卡或虚拟声卡正常播放，并且系统能暴露对应 Loopback 设备，就可以尝试。未逐个实测的游戏不应标记为已兼容。

帮它变得更适合真实游戏语音

现在最需要的是不同游戏、不同设备、不同语音环境的真实反馈。打开设置里的提交反馈，模板会带上版本、音频设备、翻译服务、最近延迟和日志路径。
